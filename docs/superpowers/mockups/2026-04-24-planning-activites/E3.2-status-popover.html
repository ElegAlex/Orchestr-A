Motif obligatoire

Avantages : empreinte minimale (popover ~120px), parfait pour un planning haute densité, libellé via tooltip au survol pour ne pas surcharger l'UI.
Inconvénients : icônes seules peu explicites sans hover (risque d'erreur de clic) ; la saisie du motif force un sous-popover ou un inline-input qui peut surprendre ; apprentissage requis.

Variante C — Expansion inline dans la cellule

Le badge se transforme en inline widget directement dans la cellule DayCell, sans popover flottant. La cellule s'agrandit temporairement pour révéler les options. Moins de superposition, zéro débordement.

C4 · Motif N/A inline cellule

38 car. · min 3

Avantages : zéro superposition (pas de z-index battle), la cellule reste dans le flux, le contexte visuel est toujours visible, les chips de statut sont lisibles sans survol. Cohérent avec le pattern "edit-in-place".
Inconvénients : le planning "saute" visuellement quand la cellule s'élargit (layout shift), nécessite un CSS transition soigné ; peut être difficile sur des lignes très compactes (hauteur fixe).
